Properties

Duration

gsap.to('.cube', { duration: 2, x: 300 });

Tells gsap to animate element with class cube moving it by 300px along x axis over 2 seconds

Delay

gsap.to('.cube', { duration: 2, delay: 1, x: 300 });

Tells gsap to animate element with class cube moving it by 300px along x axis over 2 seconds, but animation starts after 1 second delay

Repeat

gsap.to('.cube', { duration: 2, delay: 1, repeat: 3, x: 300 });

Tells gsap to animate element with class cube moving it by 300px along x axis over 2 seconds, but animation starts after 1 second delay and repeats animation 3 times. Note that first animation doesn't count to repeat so (total 4), also delay is only before aniamtion start and doesn't affect repeat animations

Yoyo

gsap.to('.cube', { duration: 2, repeat: 3, yoyo: true, x: 300, });

Tells gsap to animate element with class cube moving it by 300px along x axis over 2 seconds, repeats animation 3 times. Yoyo tells to play animation backwards-forwards.

Stagger

gsap.to('.cube', { duration: 2, x: 300, stagger: 0.2 });

Tells gsap to animate all elements with class cube, but each next aniamtion is delayed by 0.2s

gsap.to('.cube', { duration: 3, x: 600, stagger: { each: 0.5, from: 2, repeat: 2, }, });

Tells gsap to animate all elements with class cube, moving them by 600px along x axis over 3 seconds. Ecah sets delay between each cube animation. from sets index of from what cube of cube array to start (index 2 means 3 cube starts, then after 0.5s 4 and 2 cube starts and so on). Repeat in starger sets repeat to start imediatly after first element ends it starts another and doesn't wait other elements.

onComplete

gsap.to('.cube', { duration: 2, x: 300, onComplete: onCompleteHandler, });function onCompleteHandler() {
   gsap.to('.cube[data-cube="7"]', { duartion: 1, backgroundColor: '#ff00ff' });
}

Tells gsap to animate element with class cube moving it by 300px along x axis over 2 seconds, and on animation complete to start new animation that changes cube color to pink over 1 second
